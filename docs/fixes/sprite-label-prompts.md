## 1. What changed

Three custom sprite labels (TUNE, REBUILD, WHISPER) integrated into the
atlas via the slicer pipeline. The breaker, whisper, and tape puzzle
interaction prompts now render with sprite labels matching the visual
style of the existing pickup/hide/interact prompts.

Each label was generated as a raw PNG with green chroma background,
processed by the existing single_chroma pipeline, and output as a
transparent PNG in assets/ui/. The atlas.json was regenerated by the
slicer (not manually edited).

## 6. Trade-offs

The 3 puzzle prompts now match the visual style of pick-up, hide, and
interact prompts. All prompts that use E key are now sprite-based except
"Press E to leave hiding spot" (text, acceptable) and "E EXIT (silent)"
(text, per Hotfix U.9 design). Converting those to sprites is deferred.

The sliced PNGs are large (1.6-1.8 MB each). They load at runtime from
the public directory. On slow connections, the label might pop in after
the key icon renders. Compressing or resizing the source art is
deferred as polish.

The slicer pipeline depends on Python with Pillow, NumPy, and
scikit-image. If the user's environment lacks these dependencies, the
slicer cannot regenerate the atlas. The sliced output is committed
alongside the atlas.json, so consumers do not need to re-run the
pipeline.
